////// Requête conditionnelle //////


Certains Types de Requête sont strictement Conditionnels, c'est-à-dire qu'ils vérifient une Valeur (en plus de celle de la Condition) supplémentaire avant de « choisir » entre deux Cibles.

exemple :
GOAL -> 37-43 = SEE -> BOB ? TP:37-42 / NAME:FANTASIO

Lorsque la Condition est vérifiée, une nouvelle Valeur doit être vérifiée : l'objet dont le nom est "Bob" est-il dans le champ de vision de l'objet actuel ?
- si oui, téléporter l’objet actuel en x = 37 et y = 42.
- sinon, renommer l'objet en « FANTASIO ».


////// Requête classique à caractère conditionnelle //////


Certains Types de Requête "classiques" peuvent s'écrire sous la forme conditionnelle, leur écriture change afin de s'adapter à ce nouveau statut.

exemple de requête classique :
TALKED -> PEEBOY = FACING -> UP

Dans cet exemple, lorsque la Condition est vérifiée, l'objet actuel regardera en haut.

exemple de requête classique écrite sous forme conditionnelle :
TALKED -> PEEBOY = FACING -> UP ? TP:38-44 / NULL

Dans cet exemple, lorsque la Condition est vérifiée, si l'objet regard en haut, alors il sera téléporté en x = 38 et y = 44, sinon, il ne se passera rien (NULL).

ATTENTION ! Tous les Types de Requête ne peuvent pas avoir une écriture conditionnelle ! Lorsqu'elle est possible, cela sera précisé.
