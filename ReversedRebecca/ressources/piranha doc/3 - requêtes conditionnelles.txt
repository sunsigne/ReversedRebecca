////// Requête conditionnelle //////


Certains Types de Requête sont Conditionnels. C'est-à-dire qu'ils vérifient une Valeur supplémentaire (en plus de celle de la Condition) avant de choisir entre deux Cibles.
Leur écriture est différente des requêtes classiques, afin de s'adapter à leur statut particulier.

exemple :
GOAL -> 37-43 = SEE -> BOB ? GOCRYING / MOVE:37-42

Lorsque la Condition sera vérifiée, une nouvelle condition doit être vérifiée : l'objet dont le nom est "Bob" est-il dans le champ de vision de l'objet ?
- si oui, effectuer l'action GOCRYING (qui est une balise comme pour GOTO)
- si non, définir le GOAL comme étant en x=37 y=42

Note : le champ de vision est valide lorsqu'une ligne droite peut être tracée entre deux objets, et que cette ligne ne rencontre aucun obstacle défini comme "opaque" : une porte et un mur sont opaques, les objets Piranha ne le sont pas.


////// Requête classique à caractère conditionnelle //////


Certains Types de Requête "classiques" peuvent également s'écrire sous la forme conditionnelle, leur écriture change donc afin de s'adapter à ce nouveau statut.


exemple de requête classique :
TALKED -> PEEBOY_YOU_HERE = FACING -> UP

exemple de requête classique à caractère conditionnelle :
TALKED -> PEEBOY_YOU_HERE = FACING -> UP ? SLOW:38-44 / NULL

Dans le premier exemple, lorsque la Condition sera vérifiée, l'objet regardera en haut.
Dans le second exemple, lorsque la Condition sera vérifiée, Si l'objet regard en haut, alors il aura pour GOAL x=38 y=44 et devra s'y rendre lentement. Sinon, il ne fera rien.


Attention ! Tous les Types de Requête ne peuvent pas avoir un caractère conditionnel ! C'est pourquoi cela sera précisé systématiquement lorsque c'est possible.
Note : TOUTES les requêtes conditionnelles ou à caractère conditionnel ont leur 2 cibles renvoyées chacune à un GOTO à cible unitaire, l'écriture compacte est donc possible.
